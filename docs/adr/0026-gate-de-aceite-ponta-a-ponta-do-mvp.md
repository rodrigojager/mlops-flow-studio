# Gate de aceite ponta a ponta do MVP

Decidimos que o MVP só é aceito quando o usuário consegue executar o fluxo completo: montar visualmente um projeto, treinar e comparar modelos, aprovar manualmente um candidato, gerar runtime FastAPI autônomo, buildar Docker Compose com Postgres, rodar smoke, fazer predição, ver observabilidade e reimportar a saída no Studio.
