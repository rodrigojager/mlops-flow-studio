# Autoria visual como interface primária

Decidimos que o MLOps Flow Studio deve montar projetos principalmente por canvas, palette e inspector, no mesmo modelo mental do Agent Flow Studio. Specs e manifestos continuam sendo contratos versionáveis, mas a experiência principal deve permitir que o usuário, com ou sem assistência de IA, gere e edite visualmente fontes de dados, pipelines, candidatos de modelo, critérios de seleção, runtime, dashboard e container.
