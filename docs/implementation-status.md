## Fora do corte MVP / Backlog futuro

- Playwright scraping já existe como endpoint controlado da Control API para uma página raiz com login por formulário opcional, crawl interno e crawl profundo controlado de mesma origem, relatório local auditável, validação OpenAPI de endpoints/operações/schemas resumidos, descritores rasos de payload, exemplos sintéticos de payload, smoke real controlado de operação com validação de request/response, aplicação de operação validada no wizard, prévia visual de importação, edição de contrato por fonte e conversão assistida do scrape em projeto MLOps black-box. Crawling irrestrito, execução autenticada de fluxos transacionais e validação completa de JSON Schema ainda ficam fora do corte atual.
- Validação com modelo SentenceTransformers real já foi exercitada em modo online/CPU com `localFilesOnly=false` (2 textos, embedding de 384 dimensões). O worker já registra plano avançado de fine-tuning BERT/GPU por artefato, incluindo device, epochs, batch, mixed precision, gradient checkpointing e guarda ambiental antes de execução pesada. Execução real prolongada de fine-tuning em GPU ainda depende do ambiente consumidor e da flag explícita.
- Importação black-box de endpoints remotos observáveis já existe como fluxo controlado por `remoteBaseUrl` e `confirmBlackBox: true`. Importação de repositório Git com `.mlops` ou `app/metadata` já existe com confirmação explícita. Importação estática de Git sem contrato MLOps existe quando há OpenAPI, Dockerfile com labels MLOps, Compose com labels MLOps, gRPC/GRPC-gateway, servidor HTTP legado ou rotas FastAPI/Flask/Starlette/Django/Express/Fastify/Koa/Hono/NestJS/Next.js/Go/Ruby/Java/ASP.NET Core/PHP detectáveis, sem executar código externo por padrão, e também há fallback black-box genérico confirmado para Git sem sinais estáticos, com endpoint `/predict` apenas como placeholder editável. Quando habilitado por ambiente e confirmação explícita, Git/Dockerfile pode enriquecer endpoints por probe OpenAPI sandboxado sem executar entrypoint nem servidor real. Importação de imagem Docker por `docker image inspect` já existe como black-box sintético com labels MLOps quando disponíveis, e pode enriquecer endpoints por probe OpenAPI sandboxado opcional. Engenharia reversa profunda de runtimes sem endpoints observáveis e inferência automática por subir o runtime continuam fora do escopo.
- Automação opcional Prefect/Celery já é gerada junto do runtime como wrappers, manifesto e overlay `docker-compose.orchestration.yml` com Redis persistido, worker Celery e servidor Prefect local opcional por profile. Os wrappers foram validados com pacotes Prefect/Celery reais em venv isolada contra endpoints HTTP locais. Operação produtiva com priorização avançada e deploy remoto de fluxos continua pendente por depender da infraestrutura consumidora.
